# Ecosystem Alignment Pointer

FIP means Fractured Identity Protocol.

FIP remains the generic governed identity/context/object reconstruction protocol. It is not reduced to a filesystem, PSA, StarBridge, AAIR, cArla, or any one deployment.

FIPFS is the FIP-powered filesystem / virtual-filesystem embodiment of FIP.

FIP Vault is the near-term governed object vault prototype target.

PSA means Product Support Authority. PSA is the living test harness for governed workflow behavior, source/view policy, provider exposure control, proposal lifecycle trace, and StarBridge preview integration.

Standalone FIP owns the generic object, shard, metadata, authority, provenance, and reconstruction model. FIP core should support arbitrary deployment profiles and view labels.

Do not hardcode cArla/PSA/StarBridge/AAIR-specific view names into FIP core. Keep FIP generic enough for other products and deployment profiles.

Prefer authorized reconstruction planning before manifest persistence or CRUD expansion. Manifest persistence and object CRUD should follow the reconstruction/authority model, not define it prematurely.
